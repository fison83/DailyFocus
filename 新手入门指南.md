## 什么是 Vibe Coding？

**Vibe Coding** = 用自然语言 + AI 来编程

### 传统编程 vs Vibe Coding

| 传统编程 | Vibe Coding |
|---------|-------------|
| 需要学习编程语法 | 用中文/英文描述需求 |
| 自己写每一行代码 | AI 帮你写代码 |
| 遇到 bug 自己查 | AI 帮你调试 |
| 需要几个月学习 | 几分钟就能上手 |

### 一个简单的例子

**你想让应用支持夜间模式：**

```
传统方式：
1. 学习 CSS 变量
2. 编写 dark-mode.css
3. 修改 HTML 添加切换按钮
4. 编写 JavaScript 逻辑
5. 调试...

Vibe 方式：
你："帮我加个夜间模式，点击右上角按钮切换"
AI：[自动完成所有代码修改]
```

## 运行项目

### 方法一：直接打开（最简单）

1. 找到 `index.html` 文件
2. 右键 → "打开方式" → 选择浏览器（Chrome/Edge）
3. 开始使用！

### 方法二：用 VSCode 预览（推荐）

1. 在 VSCode 中打开整个项目文件夹
2. 安装 "Live Server" 扩展
3. 右键 `index.html` → "Open with Live Server"
4. 浏览器会自动打开，修改代码后自动刷新

### 依赖说明

**本项目不需要安装任何依赖！**

- 没有 `node_modules`
- 不需要 `npm install`
- 纯前端项目，打开即用

### 主要功能

| 功能 | 说明 |
|-----|------|
| 📥 **收集箱** | 快速记录任务，稍后整理 |
| 🎯 **四象限** | 按重要/紧急分类管理任务 |
| 📅 **日历** | 按日期查看和管理任务 |
| 📋 **全部任务** | 统一管理所有任务，支持批量操作 |
| ♻️ **回收站** | 已删除的任务，可以恢复或永久删除 |

### 导航说明

**电脑端：**
- 左侧边栏：点击切换不同视图

**手机端（≤1024px）：**
- 顶部导航栏：点击图标切换视图
- 左侧边栏自动隐藏

### 四象限说明

| 象限 | 含义 | 示例 |
|-----|------|------|
| 紧急且重要 | 立即处理 | 客户投诉、紧急 bug |
| 重要不紧急 | 计划处理 | 学习、锻炼、项目规划 |
| 紧急不重要 | 快速处理 | 回复邮件、日常会议 |
| 不紧急不重要 | 尽量不做 | 刷手机、无目的浏览 |

## Git 版本管理

### 什么是 Git？

**Git** = 代码的时间机器

- 每次提交都是一个"存档点"
- 可以随时回到任何历史版本
- 可以查看每次改了什么

### 基础操作

#### 1. 初始化仓库（首次）

```bash
git init
git add .
git commit -m "第一次提交：初始版本"
```

#### 2. 修改代码后保存进度

```bash
git add .
git commit -m "添加夜间模式功能"
```

#### 3. 查看历史

```bash
git log  # 查看所有提交记录
```

#### 4. 恢复到某个版本

```bash
git log  # 找到想恢复的版本的 ID（如 abc123）
git reset abc123  # 回到那个版本
```

### 提交信息规范

好的提交信息应该清晰描述改了什么：

```
✅ "添加夜间模式切换功能"
✅ "修复删除任务无效的问题"
✅ "优化手机端导航栏布局"

❌ "更新"
❌ "fix"
❌ "改动"
```

### 推荐工作流

```
1. 开始新功能前
   → git checkout -b feature/new-feature  # 创建新分支

2. 开发过程中
   → git add .
   → git commit -m "描述改了什么"

3. 功能完成后
   → git checkout main  # 回到主分支
   → git merge feature/new-feature  # 合并功能
```

## 常见问题

### Q1: 修改代码后浏览器没有变化？

**A:** 清除浏览器缓存或强制刷新（Ctrl + Shift + R）

### Q2: 出错了怎么办？

**A:**
1. 打开浏览器开发者工具（F12）
2. 切换到 Console 标签
3. 看看有没有红色的错误信息
4. 把错误信息告诉 AI

### Q3: 怎么知道改了哪个文件？

**A:** VSCode 左侧文件树会显示修改的文件（有 M 标记）

### Q4: 可以撤销修改吗？

**A:**
- 没提交：`git checkout .` 恢复所有修改
- 已提交：`git reset` 回到上一个版本

### Q5: AI 改错了怎么办？

**A:**
- 告诉 AI："不对，恢复之前的代码"
- 用 Git 回退到之前的版本
